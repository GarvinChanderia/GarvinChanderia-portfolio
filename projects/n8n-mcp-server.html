n8n MCP Server.

Giving Claude hands — a custom TypeScript bridge to a self-hosted automation engine.

The Idea

Claude is brilliant at reasoning but can't act on the real world. n8n is brilliant at executing workflows but can't reason. The n8n MCP Server connects the two — a custom TypeScript Model Context Protocol server that gives Claude Desktop direct control over a self-hosted n8n instance at localhost:5678.

How It Works

The MCP server exposes n8n's API as structured tools that Claude can call: list workflows, read workflow definitions, inject nodes, activate/deactivate, and trigger executions. Claude can now build, debug, and deploy entire automation workflows through natural conversation.

Technical Details

  • Protocol: Model Context Protocol (MCP) — Anthropic's standard for tool use.
  • Language: TypeScript, running as a local stdio server.
  • Workaround: n8n's API has a bug preventing workflow creation — the server uses a create-in-UI-then-inject pattern instead.
  • OAuth2: Gmail integration requires one-time manual credential setup before automation activates.

Powers

This server is the backbone of two major automation projects — the PM Outreach Engine and the Dental Revenue Engine. It enables Claude to orchestrate multi-step workflows: lead discovery → personalised outreach → follow-up scheduling → Telegram alerts — all from a single conversation.

Role: Solo Builder
