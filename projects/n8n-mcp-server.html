n8n MCP Server.

A custom TypeScript server that gives Claude direct operational control over a self-hosted n8n automation instance — bridging natural language reasoning with real-world workflow execution.

Problem

LLMs reason well but cannot act. Workflow engines execute reliably but cannot reason. The gap between "understanding what to do" and "actually doing it" is where most AI automation projects stall. This server closes that gap for my entire automation stack.

Implementation

Built as a Model Context Protocol server in TypeScript, running locally via stdio. It exposes n8n's REST API as structured tools: workflow listing, definition inspection, node injection, activation control, and execution triggering. Claude Desktop connects to the server and can build, debug, and deploy multi-step automation workflows through conversation.

Technical Decisions

  • Workaround architecture: n8n's API has a known bug preventing workflow creation via POST. The server uses a create-in-UI-then-inject pattern — Claude identifies the blank workflow, then populates it programmatically.
  • Credential separation: OAuth2 credentials (Gmail, Google APIs) require one-time manual setup before Claude can activate them — a deliberate security boundary.
  • Local-first: Runs at localhost:5678, not reachable from cloud sessions. This is by design — sensitive automation stays on-premise.

What It Powers

This server is the execution backbone for the PM Outreach Engine, Dental Revenue Engine, and the Nightly Distillation workflow. It enables Claude to orchestrate multi-step pipelines — lead discovery, personalised outreach, follow-up scheduling, Telegram alerting — entirely through natural language.

Role: Sole builder

Stack: TypeScript, Model Context Protocol, n8n REST API, Node.js

Status: Shipped — in daily production use
